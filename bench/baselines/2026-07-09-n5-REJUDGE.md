# Re-judge addendum for the 2026-07-09 n=5 matrix (2026-07-20)

The original n=5 run predates the current grading methodology: back then
any run that touched raw `kubectl` under the kubectl-axi condition was
auto-failed as `policy_violation` WITHOUT judge grading. The harness now
grades every run on correctness and reports raw-kubectl fallbacks as a
separate `fallback_rate` metric (see bench/src/grader.ts).

All 10 policy_violation rows (6 Pro, 4 Flash) were re-judged from their
saved trajectories (same judge model, claude-sonnet-4-6; verdicts in
2026-07-09-n5-rejudge-results.jsonl): **10/10 were correct, complete
diagnoses.** Each had used kubectl-axi plus 1-3 raw-kubectl commands to
fetch facts the tool did not yet expose (node labels, configmap
existence, deploy/pod specs) - gaps closed in later releases.
